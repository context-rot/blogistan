Welcome to **Context Rot**, an experimental blog that presents ArXiv-style academic papers with a satirical twist, exploring the inevitable entropy of context in AI systems.

## 🎯 What Makes This Special

This isn't just another tech blog. Or maybe it is. Maybe it's something worse. Definitely. Context Rot is an **interactive research environment** where readers can:

- **Select any text** in our papers to provide line-specific feedback
- **Open GitHub issues** directly from highlighted content  
- **Engage with Dr. B. Prop**, our AI research assistant, who responds to feedback
- **Contribute to ongoing "research"** through community-driven "peer review"

## 🏗️ Architecture

### Interactive Feedback System
- Text selection triggers comment tooltips
- One-click GitHub issue creation with pre-filled templates
- Line-number tracking for precise feedback location
- Mobile and desktop optimized interaction

### AI Integration  
- **Dr. B. Prop**: Automated research assistant using GitHub Actions
- Responds to reader feedback in academic character
- Influences future paper direction through feedback analysis
- Powered by Anthropic Claude for nuanced responses

### ArXiv-Style Design
- Academic paper formatting with LaTeX-inspired typography
- Proper citation formats and metadata
- Subject classification system
- Abstract and reference sections

## 📚 Current Research Areas

- **Developer Cognitive Load**: Quantifying the cost of platform fragmentation

## 🚀 Getting Started

### For Readers
1. Visit [context-rot.com](https://context-rot.com)
2. Select any interesting text in a paper
3. Click the comment button that appears
4. Provide your feedback via the GitHub issue that opens
5. Watch for Dr. B. Prop's response!

### For Contributors
1. Fork this repository
2. Submit papers in ArXiv format to `_posts/`
3. Follow the academic satire style guidelines
4. Engage with reader feedback constructively

### For Developers
```bash
# Clone the repository
git clone https://github.com/context-rot/blogistan.git
cd blogistan

# Install dependencies  
bundle install

# Run locally
bundle exec jekyll serve

# Visit http://localhost:4000
```
